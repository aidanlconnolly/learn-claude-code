# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the project

Open `index.html` directly in a browser — no server, no build step. The archived previous version is `index_v1_archive.html`.

## Architecture

Everything lives in a single `index.html` (~1500 lines). The file is divided into three logical sections:

**CSS** — all styles are inline in `<style>`. Two layout modes are styled independently: `#roadmap-view` (the skill path) and `#lesson-view` (the paginated lesson player). The color system uses CSS variables (`--navy`, `--gold`, `--green`, etc.).

**HTML** — just two `<div>` containers (`#roadmap-view` and `#lesson-view`). All content is generated by JavaScript into `#skill-nodes` and `#lesson-content`.

**JavaScript** — organized into clearly delimited sections (marked with `// ═══...`):

| Section | What it does |
|---|---|
| `SKILLS` array | All course content — 8 skills × 3 lessons × 3–4 pages. Each page has a `type` (`info`, `terminal`, `code`, `quiz`) and type-specific fields. |
| State & localStorage | `getProgress()` / `saveProgress()` / `markLessonDone()` — persists lesson completion as `{ skillId: { done: [bool, bool, bool] } }` under key `ccl_journey_v1`. |
| Roadmap render | `renderRoadmap()` — builds skill node cards dynamically from `SKILLS` and current progress. Called on init and after returning from a lesson. |
| Lesson view | `openSkill()` → `showLessonView()` → `renderLessonView()` → `renderPage()`. Navigation state is three globals: `curSkill`, `curLesson`, `curPage`. |
| Page renderers | `renderInfo()`, `renderTerminal()`, `renderCode()`, `renderQuiz()` — each returns an HTML string injected into `#lesson-content`. |
| Quiz logic | `answerQuiz(optionIdx)` and `nextQuizQuestion()` are called from inline `onclick` attributes. The quiz blocks the Next button until the final question is answered correctly. |
| Terminal animation | `animateTerminal(id, lines)` — types prompt lines character by character, then appends output lines. Triggered after `renderLessonView()` for `type:'terminal'` pages. |

## Adding content

**New page to an existing lesson** — add an object to the relevant `lesson.pages` array inside `SKILLS`. Page types:
- `{type:'info', heading, body:[], tip?, keyPoint?, breakdown:[], steps:[]}` 
- `{type:'terminal', heading, intro, lines:[{prompt:bool, text}], explain?}`
- `{type:'code', heading, lang, code, explain?}`
- `{type:'quiz', heading, questions:[{q, options:[], correct:int, fb}]}`

**New lesson or skill** — follow the same structure. Quiz pages automatically block the Next button; all other page types allow free navigation.

**Terminal `lines` format** — `{prompt: true, text: 'command'}` renders as `$ command` with typing animation. `{prompt: false, text: 'output'}` renders as grey output text appended immediately.
